### 中文: 社区 / 即刻 / V2EX / 少数派

```text
做 AI Coding 的时候，我越来越明显地感受到一个问题：

Agent 变强了，但仓库本身没有记忆。

很多关键知识最后都散落在：
- 聊天记录
- 一次性的任务上下文
- 不成体系的零散文档
- 人脑里的隐性经验

所以我做了 RepoMem。

它不是向量数据库，也不是另一个 docs 文件夹。
它是一个给代码仓库和 AI Agent 使用的持久化记忆层。

RepoMem 主要做 4 件事：
- 保存长期架构知识
- 保存长期工程记忆
- 保存版本规划
- 保存任务范围内的临时工作记忆

它最核心的机制是：
temp -> merge -> persist

也就是：
- 任务中的笔记先进入 temp
- 真正值得长期保留的知识，再通过 HITL 审阅合并进 persist
- 仓库的长期事实不直接被一次任务上下文污染

我希望它解决的问题是：
让仓库不再依赖脆弱的对话式记忆。

如果你也在用 Claude Code、Codex、Cursor 或其他 agent 做开发，RepoMem 可能会有参考价值。

项目地址：https://github.com/JasonJarvan/RepoMem
```
